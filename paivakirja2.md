# Oppimispäiväkirja: Hajautettu git

__Mikä osion tehtävissä oli vaikeaa ja mikä helppoa? Mikä auttoi minua oppimaan? Miten selvitin esteet, jotka vaikuttivat tehtävän suorittamiseen?__

Osio oli lyhyt ja nopea. Otin edellisessä osassa jo hieman varaslähtöä tehtävään Oppimispäiväkirjan osalta, niin tallennetun versionhallinnan vieminen etärepositorioon toimi tällä kertaa kertauksena. Ongelmia ei ilmennyt. Versionhallinnan vieminen etärepositorioon vei ainoastaan master-haaran, kun taas paikallisessa repositoriossa on edelleen master-haaran lisäksi tyylit-haara. Loin web-käyttöliittymässä README.md -tiedoston, ja tiedoston hakeminen paikalliseen repositorioon kävi helposti.

## Osiossa käyttämäni Git-komennot

| Komento | Kuvaus |
| --------| ------ |
| git remote | Näyttää etärepositorioiden nimet ja URL-osoitteet. |
| git push | Vie haaran viimeisimmän versionhallintaan tallennetun version etärepositorioon.  |
| git branch | Näyttää kehityshaarat tai luo uuden kehityshaaran (git branch haaran nimi). |
| git fetch | Hakee etärepositorion uusimmat muutokset paikalliseen repositorioon. |
| git checkout | Tehtävän yhteydessä vaihtaa haaraa. |
| git merge | Yhdistää kehityshaaran toiseen. |
